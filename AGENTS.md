# 项目协作规则

详细规范见 [docs/codex-usage-playbook.md](docs/codex-usage-playbook.md)。

## 任务边界

- 一个实施任务只交付一个可验证目标；不要混入无关重构。
- 开始前说明修改范围、数据库/部署影响和验证方式。
- 不与其他任务并发修改同一文件、数据库对象、任务配置或部署配置。

## Git

- 默认使用一个普通功能分支；只有真实并行且互不冲突时才使用 worktree。
- 合并、切换、清理前先检查工作区状态；已合入任务应清理临时分支/worktree。
- 不推送、部署、重启服务、删除数据或强制删除分支，除非用户明确授权。

## 数据与运行任务

- 修改数据库、同步、发布或调度前，先做只读检查并说明影响范围。
- 发布类任务不得覆盖已发布版本；失败必须保留审计和可追溯原因。
- 长任务的真实状态应写入系统审计/后台，不以对话进度代替监控。

## 验证与交付

- 修改后运行与范围相符的测试，并报告实际结果。
- 提交前运行 `git diff --check`；完成时说明改动、验证、Git 状态及后续人工动作。
- 涉及表结构、数据口径或运行方式变化时，同步更新相关 `docs/`。
