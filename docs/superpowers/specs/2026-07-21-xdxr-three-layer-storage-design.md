# XDXR 三层存储与研究复权使用设计

## 目标

把 XDXR 公司行为从“可被覆盖的一张原始表”拆为可审计的版本日志、当前事实投影和研究运行快照三层。研究复权能够精确解释每个事件版本、每个原始日线和每个因子来自何处，而线上策略读取口径保持不变。

## 非目标

- 不替换现有线上策略的 `AdjustmentService`、`DataAggregator` 或价格读取口径。
- 不在此阶段生成一张全市场物化调整价格表；价格继续由研究读取器在同一发布运行的原始快照与因子上计算。
- 不修改 Mootdx 上游接口协议。

## 三层模型

### 1. 事件版本日志：`mootdx_xdxr_event_versions`

Mootdx 只支持按股票返回完整 XDXR 事件集，不能按日期增量请求。每次成功抓取后，按事件业务键和内容指纹与该事件最新版本比较：新事件或内容变化才追加不可变版本行；内容相同只记录观察结果，不重复写入版本。版本主标识包含 `ingest_seq`、`symbol`、`event_date`、`category` 和稳定的事件标识；保留所有业务字段、`raw_json`、抓取时间与内容指纹。该表只追加，不使用 ReplacingMergeTree 覆盖历史。

`NULL` 是业务事实的一部分：新版本把 `fenhong` 等字段修正为空时，版本日志必须如实保存空值，绝不回退旧值。

### 2. 当前事实投影：`mootdx_xdxr_current`

当前投影是从版本日志派生的最新成功版本，只用于运营查询、数据质量检查和日常浏览。它按事件业务主键选择最大 `ingest_seq` 的完整行，使用对整个字段 tuple 的 `argMax`，保证可空字段来自同一个事件版本。

该投影可采用 ClickHouse View 或由显式刷新任务构建；首选普通 View，以避免双写和物化状态漂移。它不是研究复权的可复现输入。

### 抓取观察与事件消失：`mootdx_xdxr_symbol_observations`

每次按股票完整抓取均写入观察记录：`ingest_seq`、股票、抓取是否成功、返回事件数量、事件业务键集合指纹、耗时和错误。某次成功抓取中旧事件没有出现时，系统只记录“未观察到”，不删除事件版本、不切换当前事实。连续成功观察缺失达到阈值后标为 `suspected_withdrawn` 并告警；只有显式确认才追加撤回版本并触发复权重算。

### 3. 研究复权运行快照

每次研究复权构建从事件版本日志中，以捕获的连续已结算 `ingest_seq` 为上界，只选择状态为 `succeeded` 的同步批次，并按事件业务主键取该上界内最新完整版本。选中的事件以原始字段、来源 `ingest_seq`、内容指纹与校验结果写入 `research_adjustment_events`。

同一运行还保存原始日线快照与日级因子；发布前要求原始日线和因子逐标的、逐日期完整对应。读取器只能读取当前发布运行自己的原始日线快照、事件审计和因子。

## 同步与版本边界

1. Mootdx 同步分配全局 `ingest_seq`，先写 `running` 审计记录。
2. XDXR 成功抓取写入股票观察记录；只有新增或内容变化的事件写入版本日志。运行完成后审计记录置为 `succeeded`，失败记录为 `failed`。
3. `mootdx_xdxr_current` 自动从版本日志投影当前事实。
4. 研究构建捕获连续已结算边界；仅消费 `(previous_input_ingest_seq, captured_ingest_seq]` 内的成功批次作为变化范围，但重算目标标的完整历史时可读取不超过捕获边界的所有成功版本。
5. 失败批次可推进已结算边界，但不提供事件输入。

## 迁移策略

- 新建版本日志、观察表和当前投影，不重写或依赖旧 `mootdx_xdxr` 的 ReplacingMergeTree 历史。
- 将旧表原子重命名为带迁移批次后缀的只读备份，而不是直接删除；创建兼容视图 `mootdx_xdxr` 指向新的当前事实投影，逐步迁移现有读路径。
- 新表不从旧表伪造版本历史。首次基线通过全股票池完整 XDXR 抓取建立，所有基线事件使用该次真实 `ingest_seq`；这轮已基准验证约 5200 股票、约 253 秒。
- 基线完成后比较旧表当前事实与新投影的事件数量、业务键覆盖和字段差异；差异异常时不切换兼容视图。
- 后续 XDXR 同步只写版本日志和观察表；旧备份在一个已发布研究快照周期后才允许人工清理。
- 研究复权首次切换版本日志后必须 `--full` 发布一个新运行；旧研究版本继续可读，但不能作为新增量的事件基线。

## 校验与失败语义

- 每个事件版本必须有稳定内容指纹；同一批次内完全相同版本可去重，但不同批次的版本历史仍可审计。
- 研究事件选择使用整行 tuple 的 `argMax`，不会把不同版本的字段混合，也不会忽略最新 `NULL`。
- 原始事件、日线或因子缺失时，候选运行不发布，上一已发布运行继续可读。
- 版本日志只追加；当前投影或研究构建失败不影响历史版本。

## 验收标准

- 同一事件两次抓取均可追溯，当前投影显示较大成功 `ingest_seq` 的完整版本。
- 新版本将某个可空业务字段改为 `NULL` 时，当前投影和研究快照均保留 `NULL`。
- 同一内容被重复抓取不新增事件版本，但保留一条股票观察审计。
- 一次事件消失只产生未观察记录；没有显式确认不得删除或撤回当前事件事实。
- 捕获序号之前的事件版本进入本次研究快照，之后的版本不会混入。
- 失败同步不进入当前投影的成功版本选择，也不进入研究输入。
- 同一研究运行可通过 `run_id` 追溯事件版本、原始日线和因子；线上策略路径无代码行为变化。
