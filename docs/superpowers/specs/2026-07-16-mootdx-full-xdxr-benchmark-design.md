# Mootdx 全量 XDXR 基准设计

## 目标

建立可重复的全股票池、只读 Mootdx XDXR 基准，并将一次真实运行的结果保存为可审阅报告。基准用于衡量实际耗时、成功率、延迟分位数和失败分布；不向 ClickHouse 写入事件，也不改动任何复权价格或策略逻辑。

## 命令与运行边界

保留 `scripts/benchmark_mootdx_xdxr.py` 的默认 300 标的分层抽样行为。新增显式 `--all` 开关：它选择已读取目录的全部可用标的；`--all` 与 `--sample-size` 互斥，避免操作者误把抽样当全量。

基准继续默认只读，且禁止 `--all --write` 组合。全量网络运行不应以写入事实表为副作用；后续正式 bootstrap 另行决策。

新增 `--output` 可选路径。指定后，将与标准输出完全相同的 JSON 写入该路径，并自动创建父目录。报告文件名由运行命令明确指定，便于按日期归档。

## 指标与失败分布

报告继续提供目录/样本规模、成功/空/错误数、事件数、总请求耗时和 P50/P95/P99 延迟。只读模式另增加：

- `bucket_results`：每个市场/ST 分层桶的请求数、成功数、空结果数、错误数和事件数；
- `error_types`：异常按 `异常类型: 截断消息` 聚合的次数；
- `failed_symbols_sample`：按请求顺序保留最多 20 个 `{symbol, bucket, error}` 样本。

每个请求无论成功、空结果或抛错都计入延迟分位数，确保长尾指标表示真实调用体验。空结果不是错误，单独统计。

## 验收

- 默认 300 标的命令与既有行为兼容；新增字段不改变旧计数含义。
- `--all` 选择全部目录标的，且不能与 `--sample-size` 或 `--write` 同时使用。
- 可注入的读失败被正确聚合到市场桶、错误类型和有限样本。
- `--output` 生成可解析 JSON，其内容与 stdout 一致。
- 使用固定 `bestip=False`、`rate_limit=0.02`、`timeout=10` 对当前全股票池运行一次，报告保存到 `reports/mootdx_probe/`；若源端失败，仍保存观察到的结果并如实报告，不重试或改参数掩盖失败。
