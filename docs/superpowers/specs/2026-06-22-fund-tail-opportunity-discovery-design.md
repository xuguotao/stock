# 基金尾盘机会发现器设计

## 背景

当前基金尾盘模块围绕固定监控清单运行，适合跟踪已持仓和少量候选基金，但不适合主动发现新的尾盘机会。用户希望利用历史交易数据、代理指数和当天行情，从更大的基金池中筛出更适合当前尾盘策略的基金，而不是只盯着现有 16 只。

## 目标

把基金尾盘模块从“监控固定基金”升级为“发现可操作基金机会”。系统每天在尾盘前从更大候选池中筛选基金，输出适合新开仓、加仓、持有观察、减仓和排除的名单。

第一版目标不是全市场无差别扫描，而是建立一个高质量候选池，优先保证数据可信、代理匹配有效、交易费率适合用户的尾盘策略。

## 非目标

- 不承诺预测一定准确或保证收益。
- 不直接替代用户的最终交易决策。
- 第一版不引入复杂机器学习模型，先复用现有可解释规则和历史相似样本回测。
- 第一版不覆盖货币基金、纯债基金、封闭期基金和长期持有期明显不适合尾盘策略的产品。

## 候选池范围

候选池分三层：

1. 优先池：宽基指数、行业指数、ETF 联接、LOF、QDII 指数类基金。这类产品和指数代理关系更清晰，盘中估值更可靠。
2. 谨慎池：主动混合、主动股票基金。只有代理匹配度、历史相关性、风格稳定性达标时才进入可操作名单。
3. 排除池：货基、纯债、封闭期、持有期过长、规模过小、暂停申购赎回、费率不适合短线、数据缺失严重的基金。

第一版候选数量建议控制在 100-300 只，避免全市场数据噪音淹没有效信号。

## 数据来源

系统复用现有 ClickHouse 和 CSV 流程，并新增基金候选池数据：

- 基金基础信息：基金代码、名称、类型、成立时间、规模、申购赎回状态、费率标签。
- 历史净值：用于计算基金自身历史收益和回撤。
- 代理指数：用于尾盘实时估算和历史相似场景回测。
- 当天代理行情：用于生成今日尾盘信号。
- 用户观察池/持仓池：用于区分“新开仓候选”和“已持仓加减仓建议”。

第一版可先使用静态候选池配置文件或导入 CSV，后续再接入自动基金列表下载。

## 代理匹配

每只基金必须绑定一个代理标的。绑定方式分两步：

1. 规则初选：根据基金名称、类型和主题关键词匹配代理指数，例如沪深300、中证500、创业板、消费、医疗、新能源、环保、纳指/QQQ。
2. 历史验证：用基金 NAV 日收益和代理指数日收益计算相关性、样本数和误差。

只有代理匹配等级为高或中，并且样本数足够的基金，才允许进入可操作候选。代理匹配低的基金只进入观察，不给尾盘买入建议。

## 筛选与评分

机会发现器输出一个综合评分，排序规则以未来 3-5 个交易日的概率和风险为核心。

主要评分因子：

- 今日代理涨跌率和历史分位。
- 近 3/5/10/20 日动量。
- 距离 20 日均线的位置。
- 近期回撤和波动率。
- 未来 1/3/5/10 日相似样本上涨概率。
- 未来 3/5 日中位收益。
- 跌超 1%/2% 概率。
- 代理匹配度。
- 基金类型风险权重。
- 节假日、周末、长假前风险惩罚。
- 费率和持有期惩罚。

评分结果转成操作等级：

- A：重点关注，只能在非追高、数据新鲜、风险收益比明显占优时出现。
- B：小额试探，适合轻仓验证，不允许连续补仓摊低。
- C：持有观察或等待回踩，说明信号有潜力但条件不充分。
- D：不加仓或回避。

## 输出结果

页面和报告新增“机会发现”视图，和现有监控列表并存。

建议展示四组结果：

1. 今日可新开仓候选：用户未持有、评分靠前、代理可信、费率适合。
2. 今日可加仓候选：用户已持有、趋势和预测都支持继续加。
3. 今日减仓/止盈候选：涨幅过大、风险收益比变差或弱势破位。
4. 明确排除：追高、弱势、数据滞后、代理不可信、费率不适合。

列表默认按综合评分排序，同时支持按基金类型、操作等级、代理匹配度、未来 5 日上涨概率、跌超 2% 概率筛选。

## 风险控制

第一版必须内置以下保护：

- 休市日不生成新的尾盘交易建议，只生成观察报告。
- 长假前最后一个交易日自动降低新开仓等级。
- 高波动行业基金如果只是 B 级，不允许给“重点加仓”措辞。
- 涨幅历史分位过高时，买入建议降级，已有盈利仓可提示小比例落袋。
- 代理匹配低或数据滞后时，不给买入建议。
- 新开仓基金必须显示费率和建议最短观察周期。

## 后端改造

新增一个纯研究模块，负责候选池筛选和评分：

- 读取候选基金池。
- 拉取或读取基金 NAV、代理指数和当天行情。
- 自动计算代理匹配。
- 复用现有 `fund_tail_backtest` 的信号、预测和卖出逻辑。
- 输出候选评分表和中文报告。

现有固定监控清单继续保留，用于用户自定义持仓和观察。机会发现器输出的基金可以一键加入观察池。

## 前端改造

基金尾盘页面新增 Tab：

- 最新操作建议：保留现有监控清单。
- 机会发现：展示全候选池筛选结果。
- 候选池管理：维护是否启用、基金类型、代理指数、费率标签和备注。

第一版可以先只做“机会发现”表格和“一键加入观察池”，候选池管理可先用配置文件或 CSV。

## 测试策略

需要覆盖：

- 候选基金过滤规则。
- 代理匹配评分。
- 休市日和长假前降级规则。
- 已持仓和未持仓操作建议差异。
- 高涨幅止盈提示。
- API 返回结构。
- 前端页面核心字段展示。

## 第一版交付建议

第一版做最小可用闭环：

1. 新增候选池配置文件，先人工放入 100-300 只基金。
2. 新增机会发现脚本，输出 `reports/fund_tail_opportunities.csv` 和 Markdown。
3. 在基金尾盘页面新增“机会发现”Tab。
4. 支持从机会发现结果一键加入当前观察池。
5. 暂不做全市场自动基金发现，等评分稳定后再扩展。

## 验收标准

- 每天尾盘前能生成候选基金排序。
- 输出结果能区分新开仓、已持仓加仓、减仓和排除。
- 每个建议都有中文原因、关键概率、风险指标和代理匹配可信度。
- 休市日不会生成误导性的当天交易建议。
- 用户能从候选结果中把基金加入观察池。
