# Mootdx 日线缺口按交易日查询设计

## 目标

让用户在“日线质量”页面选择一个交易日，并只查看该交易日的缺失标的及其判定结果，从而能够独立抽样验证缺失是否合理。

## 范围

- 默认选中最新交易日。
- 可选择当前质量窗口内的任一交易日。
- 缺口工作队列随选中交易日联动，只显示该日缺失的标的。
- 选中日期的统计同时展示：物理缺失、已知无数据、建议回补、待核验。
- 保持现有分类、Baostock 核验、定向回补和任务进度能力。

## 后端

`daily_quality` 在现有 `daily_coverage` 每日统计中增加日期可用的分类计数。缺口明细继续按标的和连续日期块生成，前端以 `missing_dates` 是否包含选中交易日过滤，无需引入第二套查询接口。

统计分为两个口径：

- 物理缺失：目录预期标的中没有写入 `mootdx_stock_kline` 的数量。
- 处理分类：已知无数据、建议回补、待核验和人工确认无需回补。已知无数据保留在明细中，不能被当作可回补缺口。

## 前端

在“缺口工作队列”上方增加“查询交易日”选择器和最近交易日覆盖条：

- 覆盖条点击某日后更新选中日期。
- 日期选择器用于直接跳转窗口中的任一交易日。
- 队列标题显示选中日期。
- 过滤标签的数量以选中交易日为准，而非整个观察窗口。
- 缺失交易日列在单日查询时只展示选中日期，避免多日连续缺口造成阅读噪声。
- 默认仍使用最新交易日，因此首次打开即可解释顶部“最新覆盖”的缺失标的。

## 边界与错误处理

- 选择日期没有缺口时，显示空队列和该日“缺失 0”。
- 更换观察窗口后，若当前选中日期不在窗口内，自动回退至最新交易日。
- 核验与回补只接收当前日期且属于相应分类的选中项，避免把同一标的跨日期的缺口混入任务。

## 验证

- 后端测试覆盖每日分类统计与连续缺口按日期过滤。
- 前端静态测试覆盖日期选择器、队列过滤和日期联动统计。
- 构建前端并通过 API 抽验最新交易日：61 条物理缺失全部显示为已知无数据。
