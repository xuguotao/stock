# 基金尾盘预测准确率优化待办

目标：把基金尾盘建议从“规则信号”升级为“未来 N 天收益概率预测 + 风险校准”，服务于尾盘是否加仓的收益目标。

## Phase 1：可解释预测层

- [x] 为每只基金输出未来 1/3/5/10 日上涨概率、平均收益、中位数收益、跌超 1%/2% 概率。
- [x] 使用与当天相似的历史场景做条件统计，而不是只看全量 add 信号。
- [x] 将预测分数纳入 A/B/C/D 等级，防止单纯趋势强就给 A。
- [x] 对大涨日增加追高惩罚：涨幅越大，越需要更高的未来胜率和收益确认。
- [x] 中文 CSV 和 Markdown 报告展示关键预测字段。

## Phase 2：特征增强

- [x] 增加今日涨跌幅历史分位数。
- [x] 增加近 3/5/10/20 日动量。
- [x] 增加距离 5/20/60 日均线偏离度。
- [x] 增加连续上涨/下跌天数。
- [x] 增加近 10/20 日波动率和近期回撤深度。
- [x] 如果代理数据有成交量，增加成交量相对 20 日均量。
- [ ] 增加基金代理相对沪深 300 的强弱。

## Phase 3：分类型/分市场状态

- [ ] 按宽基、消费、医药、新能源、环保、科技/QDII、主动混合分类。
- [ ] 为不同基金类型设置不同的相似样本条件和追高惩罚。
- [ ] 增加市场状态识别：趋势市、震荡市、下跌反弹、缩量环境。
- [ ] QDII 单独引入纳指/QQQ、美元人民币、美债收益率等特征。

## Phase 4：回测校准

- [ ] 统计预测分数分档后的真实未来收益。
- [ ] 统计 A/B/C/D 各档未来 1/3/5/10 日实际胜率和收益。
- [ ] 检查预测 60% 上涨时，真实上涨比例是否接近 60%。
- [ ] 输出错误案例：高分但亏损、低分但上涨的集中原因。
- [ ] 用滚动窗口回测，避免未来数据泄漏。

## Phase 5：机器学习候选

- [ ] 在 Phase 1-4 稳定后，再尝试逻辑回归/随机森林/LightGBM。
- [ ] 保留可解释基线模型，机器学习模型必须跑赢基线才用于建议。
- [ ] 严格使用时间序列切分，不做随机切分。
