# TDXRS 数据源使用文档

## 概述

TDXRS 是通达信 (TDX) 行情数据的 Rust 高性能实现，通过 PyO3 提供 Python 接口。它直连通达信行情服务器（TCP 二进制协议），可以获取 HTTP API 无法提供的数据。

**版本**: 0.6.5  
**协议**: 通达信私有 TCP 协议（逆向工程）  
**服务器**: 101 台已知券商服务器（海通/广发/华林/国信等）

---

## 可获取的数据类型

### 1. K 线数据（支持 12 种周期）

| 周期 | Category 代码 | 单次最大条数 | 历史回补深度 |
|------|--------------|-------------|-------------|
| 1 分钟 | 8 | 800 条 | ~3 个交易日 |
| 5 分钟 | 0 | 800 条 | ~23 个交易日 |
| 15 分钟 | 1 | 800 条 | ~2 个月 |
| 30 分钟 | 2 | 800 条 | ~4 个月 |
| 60 分钟 | 3 | 800 条 | ~8 个月 |
| 日线 | 4 | 800 条 | ~3 年（2023-03-15 起） |
| 周线 | 5 | 800 条 | ~15 年 |
| 月线 | 6 | 800 条 | ~66 年 |

**示例数据**:
```python
bars = client.get_security_bars(4, 0, "000001", 0, 10)  # 日线，平安银行
# 返回字段: datetime, open, high, low, close, vol, amount
```

**关键发现**:
- ✅ **日线可回补 3 年**（2023-03-15 至 2026-07-03）
- ✅ **5 分钟线可回补 23 天**（2026-06-10 至 2026-07-03）
- ✅ **1 分钟线可回补 3 天**（2026-06-30 至 2026-07-03）
- ⚠️ 分钟线数据量有限，适合增量同步，不适合全量回补

---

### 2. 实时行情（五档盘口）

**数据内容**:
- 最新价、涨跌幅、成交量、成交额
- **五档买卖盘**（bid1-5, ask1-5 及对应量）
- 今开、最高、最低、昨收

**示例数据**:
```python
quotes = client.get_security_quotes([(0, "000001"), (1, "600000")])
# 返回字段:
#   code, price, change_pct, vol, amount, open, high, low, last_close
#   bid1, bid_vol1, bid2, bid_vol2, ..., bid5, bid_vol5
#   ask1, ask_vol1, ask2, ask_vol2, ..., ask5, ask_vol5
```

**实测**:
```
股票: 000001 | 最新价=10.29 | 涨幅=0.00% | 成交量=863326 | 成交额=888789376
  档位 1 | 买10.29×1518 | 卖10.30×489
  档位 2 | 买10.28×1636 | 卖10.31×2231
  档位 3 | 买10.27×395  | 卖10.32×5606
  档位 4 | 买10.26×525  | 卖10.33×1800
  档位 5 | 买10.25×476  | 卖10.34×2574
```

**价值**: 这是 HTTP API（Tencent/Sina）**无法提供**的数据维度。

### 3. 除权除息数据（XDXR）

**数据内容**:
- 除权除息日期
- 分红金额（每股）
- 送股数量（每股）
- 配股数量（每股）
- 缩股数量（每股）

**示例数据**:
```python
xdxr_list = client.get_xdxr_info(0, "000001")  # 平安银行
# 返回字段: year, month, day, category, bonus_amount, ratening_amount, increased_amount
```

**实测**:
```
平安银行 (000001) 除权除息记录数: 79
  1990-03-01 | 类型=1 | 分红=0.000 | 送股=0.000 | 配股=0.000
  1991-04-03 | 类型=5 | 分红=0.000 | 送股=0.000 | 配股=0.000
  ...
```

**关键发现**:
- ✅ **历史数据可回补到 1990 年**（平安银行上市以来所有除权事件）
- ✅ 对策略回测的复权计算**至关重要**
- ✅ 这是 HTTP API **无法提供**的核心数据

---

### 4. 逐笔成交数据

**数据内容**:
- 成交时间（精确到分钟）
- 成交价格
- 成交量
- 买卖方向（0=买, 1=卖, 2=集合竞价）

**示例数据**:
```python
transactions = client.get_transaction_data(0, "000001", 0, 10)
# 返回字段: time, price, vol, buyorsell
```

**实测**:
```
平安银行 (000001) 当日逐笔成交（前 10 笔）:
  14:56 | 价格=10.30 | 成交量=49  | 买卖方向=0 (买)
  14:56 | 价格=10.30 | 成交量=84  | 买卖方向=0 (买)
  14:56 | 价格=10.29 | 成交量=130 | 买卖方向=1 (卖)
  15:00 | 价格=10.29 | 成交量=7895 | 买卖方向=2 (集合竞价)
```

**价值**:
- 可用于大单分析、主力行为识别
- 可计算更精确的资金流向
- ⚠️ 仅当日数据，不支持历史回补

---

### 5. 证券列表

**数据内容**:
- 证券代码
- 证券名称
- 小数位（价格精度）
- 前收盘价

**示例数据**:
```python
stocks = client.get_security_list(0, 0)  # 深圳市场，从第 0 只开始
# 返回字段: code, name, decimal_point, pre_close
```

**实测**:
```
深圳市场 (market=0): 23,730 只证券
上海市场 (market=1): 27,541 只证券
北京市场 (market=2): 350 只证券
```

**注意**: 返回的不仅是股票，还包括指数、基金、债券等。需要过滤出 A 股代码。

---

### 6. 证券数量

**用途**: 获取市场总证券数，用于批量拉取列表。

```python
count = client.get_security_count(0)  # 深圳市场: 23730
```

## 性能基准

### 单次请求耗时

| 数据类型 | 数据量 | 平均耗时 | 吞吐量 |
|---------|--------|---------|--------|
| 日线 800 条 | 800 bars | 0.488s | 2.0 次/秒 |
| 5 分钟 800 条 | 800 bars | 0.494s | 2.0 次/秒 |
| 实时行情 10 只 | 10 quotes | 0.061s | 16.4 次/秒 |
| 证券列表 1000 只 | ~1000 stocks | <0.001s | 2026.6 次/秒 |

### 批量同步效率估算

假设需要全市场同步（5000 只股票）：

**场景 1: 日线同步（增量）**
- 每只股票拉取最近 10 条日线
- 单次耗时: ~0.05s
- 总耗时: 5000 × 0.05s = **250s ≈ 4 分钟**

**场景 2: 5 分钟线同步（增量）**
- 每只股票拉取最近 48 条（当日）
- 单次耗时: ~0.05s
- 总耗时: 5000 × 0.05s = **250s ≈ 4 分钟**

**场景 3: 除权除息全量同步**
- 每只股票拉取全量 xdxr
- 单次耗时: ~0.05s
- 总耗时: 5000 × 0.05s = **250s ≈ 4 分钟**

**优化手段**:
- 多服务器轮转（5 台并行）→ 吞吐 × 5
- 连接池 + 心跳保活
- 自适应限流（盘后放宽到 4×）

## 推荐使用场景

### ✅ 适合使用 TDXRS 的场景

1. **获取五档盘口数据** — HTTP API 无法提供
2. **获取除权除息数据** — 复权计算必需
3. **高频实时行情** — 毫秒级延迟，适合盘中监控
4. **逐笔成交分析** — 大单识别、主力行为
5. **网络不稳定环境** — TCP 长连接比 HTTP 更稳定

### ❌ 不适合使用 TDXRS 的场景

1. **全量历史分钟线回补** — 数据深度有限（5m 仅 23 天）
2. **日线全量回补（>3 年）** — 数据深度有限
3. **需要完整财务数据** — TDXRS 的财务接口返回有限

### 💡 最佳实践

**数据源优先级**:
```
ClickHouse (本地权威)
  → TDXRS (五档/xdxr/逐笔/实时)
    → Tencent (日线/分钟线/历史)
      → Sina (兜底)
```

**同步策略**:
- **盘中**: 用 TDXRS 采集五档盘口 + 逐笔成交（实时）
- **盘后**: 用 TDXRS 同步除权除息（每日一次）
- **历史回补**: 用 Tencent/Sina（数据深度更大）

## 限制与注意事项

1. **分钟线数据深度有限**: 5m 仅 23 天，1m 仅 3 天。不适合全量历史回补。
2. **逐笔成交仅当日**: 不支持历史逐笔数据。
3. **需要网络连接**: 必须连接通达信服务器，不能离线使用。
4. **服务器稳定性**: 依赖券商服务器，高峰期可能限流。
5. **协议逆向**: 通达信协议是逆向工程实现，可能随版本更新而变化。

---

## 总结

TDXRS 的核心价值在于提供 **HTTP API 无法获取的数据**：
- ✅ 五档盘口（实时）
- ✅ 除权除息（全量历史）
- ✅ 逐笔成交（当日）

对于日线、分钟线等常规数据，建议继续使用 Tencent/Sina（历史深度更大）。TDXRS 作为补充源，填补 HTTP API 的数据空白。
