# 后台可视化升级路线图

## 当前定位

后台是本地量化研究与模拟交易的操作台。它负责把已有 CLI 和 `src/` 业务能力可视化，不在前端复制策略、数据下载或回测逻辑。

## 已完成

- 总览：展示任务概况和最近任务。
- 数据中心：扫描 `data/research/*.parquet`，查看 manifest、日期范围、行数、符号数和符号列表。
- 尾盘回测：支持样例数据，也支持从数据中心选择 `dataset_id` 运行本地 research dataset 回测。
- 基金尾盘：查看基金池数据状态、读取最新建议报告、基于本地 CSV 生成建议任务。
- 任务中心：SQLite 持久化任务状态、参数、结果、错误和时间戳。

## 下一阶段顺序

1. 任务中心增强
   - 查看单个任务详情、参数、结果和错误。
   - 支持按任务类型、状态过滤。
   - 对长结果提供折叠、复制和下载。

2. 尾盘回测增强
   - 回测参数面板化：市场宽度、min score、top N、资金、日期范围。
   - 回测结果增加交易列表、持仓变化、指标格式化。
   - 支持从历史任务恢复结果视图。

3. 尾盘 live 扫描后台化
   - 复用 `src.strategy.tail_session.live`。
   - 提供股票池选择、扫描参数、选股结果和日报查看。
   - 初期只做本地/手动运行，不接实盘下单。

4. 基金尾盘增强
   - 增加基金选择、多基金过滤、报告导出。
   - 明确本地生成和联网下载的边界。
   - 联网下载应作为单独任务加确认，不混入普通查看请求。

5. TimesFM 和 Qlib 融合
   - TimesFM 先做特征文件浏览、任务触发和结果对比。
   - Qlib 先作为研究模型适配器：导入/导出数据、读取预测分数、接入现有回测。
   - 不用 Qlib 替换现有 A 股交易、风控和尾盘分钟扫描。

6. 运维化
   - 后台任务从进程内 BackgroundTasks 升级为 worker/queue。
   - 增加任务日志、取消、重试和执行环境记录。
   - 增加权限/确认机制，避免误触联网下载或模拟交易。

## 边界原则

- 前端只提交参数和展示结果。
- 后端 Web 层只做编排和路径解析。
- 策略、数据、研究逻辑继续沉在 `src/`。
- CLI 仍保留，Web 与 CLI 应复用同一业务函数。
