# 统一可用股票池标签设计

## 目标

建立全项目统一的可用股票池标签层，供 Mootdx 目录质量、日线同步、策略选股、回测和其他数据源模块共同使用。页面统计直接读取最新标签快照，不再临时扫描日线表计算流动性和有效性。

## 设计原则

- 原始目录和日线表保存事实；标签表保存可重算的派生判断。
- 约 5,000 只标的、20 个交易日的计算量很小，首期采用单任务全量重算，避免多任务触发链。
- 事实标签固定，业务阈值通过后台配置。
- 标签计算失败时保留上一版有效快照，不覆盖为不完整结果。

## 标签快照

新增 `stock_universe_profiles`，按 `symbol` 保留最新版本；以 `computed_at` 作为 ReplacingMergeTree 版本列。

| 分类 | 字段 | 含义 |
| --- | --- | --- |
| 标识 | `symbol`, `as_of_date`, `computed_at`, `rule_version` | 标的、标签所依据的交易日、计算时间和规则版本。 |
| 目录事实 | `market`, `is_st`, `list_date`, `listing_age_days` | 市场、ST、上市日期与上市天数。 |
| 目录判断 | `catalog_valid` | 沪深 A 股、非 ST、正常上市。 |
| 日线判断 | `latest_daily_valid` | `as_of_date` 存在合法日线。 |
| 流动性事实 | `recent_20d_bar_count`, `recent_20d_trading_days`, `recent_20d_avg_amount`, `recent_20d_median_amount`, `recent_20d_zero_volume_days` | 最近窗口的可用日线、成交天数、成交额和零成交统计。 |
| 流动性判断 | `liquidity_qualified`, `liquidity_level` | 是否达到阈值及流动性分层。 |
| 最终判断 | `universe_eligible`, `exclusion_reasons` | 是否纳入默认可用池，以及全部排除原因。 |

默认规则：近 20 日至少 15 个有成交日，日均成交额至少 1,000 万元。

## 单任务刷新

新增唯一任务 `stock_universe_profile_refresh`：

- 交易日 `16:15` 运行，在 `mootdx_daily_kline_reconcile` 后。
- 每次全量计算当前有效目录中的全部标的。
- 使用最新可用交易日作为 `as_of_date`，读取统一日线事实表；日线可来自 mootdx 或 Baostock。
- 手工运行支持 `symbols` 参数，只重算指定标的，用于排障；实现仍复用同一个核心计算函数。
- 写入运行审计：目录总数、目录有效数、日线有效数、流动性达标数、最终可用数、规则版本和失败样本。

catalog、日线或 Baostock 核验完成后不立即触发额外任务；页面可标记标签相对最新事实是否待刷新，等待 `16:15` 的统一全量计算。

## 规则配置

后台仅开放业务阈值：

```text
lookback_days = 20
min_trading_days = 15
min_average_amount = 10_000_000
min_listing_age_days = 0
include_beijing = false
```

固定事实标签不开放修改：市场归属、ST、上市状态、日线是否存在、日线 OHLCV 合法性。保存配置时递增 `rule_version`，下一次刷新使用新版本；可提供“立即刷新”手动任务。

## Catalog 质量页

在现有目录质量页面新增“可用股票池筛选统计”区：

1. 漏斗：目录总数 → 目录有效 → 日线有效 → 流动性达标 → 最终可用。
2. 多选观察条件：市场、ST、上市时长、日线有效、近 20 日成交天数、日均成交额、流动性等级、排除原因。
3. 每项同时显示命中数、排除数和占比；多个条件显示交集后的标的数。
4. 分布表：按市场、上市年限、成交天数分桶、成交额分桶、排除原因统计。
5. 点击任一统计项可查看标的样本与标签明细。
6. 页面显示标签 `as_of_date`、计算时间、规则版本及是否落后于最新 catalog/日线。

## 验收标准

1. 页面查询只读取标签快照和轻量聚合，不扫描完整日线历史表。
2. 默认规则下可明确解释任意标的为何未纳入可用池。
3. 每个交易日收盘后生成一版完整、一致的标签快照。
4. 调整阈值后规则版本变化，刷新任务按新规则重算。
5. 日线同步、策略模块可通过 `universe_eligible` 与排除原因复用同一口径。
