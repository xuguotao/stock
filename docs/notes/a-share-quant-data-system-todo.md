# A 股量化交易数据体系 TODO

## 目标

建设一套面向 A 股量化研究、模拟交易和后续实盘扩展的数据体系。优先保证数据可交易、可复盘、可解释，再逐步扩展因子、基本面、宏观和执行质量数据。

核心原则：

- 先做“能不能交易”的数据，再做“为什么上涨”的数据。
- 原始数据保留，标准化数据可修正，因子数据可重建，研究样本必须可复现，交易数据必须可审计。
- 每个数据源都要有质量统计、更新机制、失败告警和修复路径。

5. 建立涨跌停、停复牌、ST、新股上市天数等交易约束表。

## 推荐数据仓分层

| 层级 | 作用 | 要求 |
| --- | --- | --- |
| `raw_layer` | 保存原始接口返回 | 不覆盖，保留来源和抓取时间 |
| `normalized_layer` | 标准化后的股票、行情、财务、事件数据 | 可修正，有主键和版本 |
| `quality_layer` | 数据完整度、异常检测、修复记录 | 可被数据中心和数据日历直接读取 |
| `feature_layer` | 因子、标签、市场状态、行业暴露 | 可重建，有版本 |
| `research_layer` | 可复现实验样本 | 锁定时间窗口、样本池和数据口径 |
| `trading_layer` | 信号、订单、成交、持仓、风控状态 | 可审计，不允许静默覆盖 |

## 阶段验收标准

### 第一阶段：数据可用

- 股票池数量和真实市场数量基本一致。
- 日线覆盖率、分钟线覆盖率、实时快照覆盖率可以按交易日查看。
- 数据中心能明确展示数据缺失、异常和最近修复情况。

### 第二阶段：回测可信

- 回测能处理停牌、涨跌停、ST、新股、T+1。
- 每次研究样本能复现。
- 策略收益可以拆解到个股、行业和风格。

### 第三阶段：模拟交易可审计

- 每个信号都有输入数据版本和生成时间。
- 每笔订单都有触发原因、成交结果和失败原因。
- 每日持仓、净值、风险暴露可回放。

### 第四阶段：实盘扩展准备

- 数据任务独立运行，不依赖后台页面。
- 数据异常会阻断交易或降级策略。
- 行情、订单、成交、持仓、风控形成闭环。

## 暂不做

- 不急于接入过多新闻、舆情、研报数据。
- 不急于建设复杂宏观数据库。
- 不急于引入高频逐笔成交和 Level-2，除非明确策略需要。
- 不在 P0/P1 未稳定前扩展大量基本面因子。
