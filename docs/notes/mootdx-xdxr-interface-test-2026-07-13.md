# Mootdx XDXR 接口测试与字段说明

测试时间：2026-07-13 23:35（Asia/Shanghai）
项目版本：`mootdx==0.11.7`
测试性质：只读小样本探测；未写入 ClickHouse、未创建定时任务。

## 结论

`mootdx` 的标准行情客户端可以稳定返回个股除权除息及股本变动历史。此次 4 只股票、每只连续 3 轮，共 12 次请求全部成功；同一标的各轮事件数一致。

这只能说明当前网络、当前通达信服务器和所选样本可用，不能作为全量任务耗时或长期稳定性的结论。首次请求包含建连成本，应与客户端复用后的稳态请求分开统计。

## 测试样本与性能

调用路径：

```python
source = MootdxSource(rate_limit=0.05, timeout=15, bestip=False)
events = source.fetch_xdxr("000001.SZ")
```

适配器底层调用：

```python
Quotes.factory(market="std").xdxr(symbol="000001")
```

| 样本 | 市场/类型 | 每轮事件数 | 成功次数 | 稳态中位延迟 | 最大延迟 |
| --- | --- | ---: | ---: | ---: | ---: |
| `000001.SZ` | 深市主板 | 79 | 3 / 3 | 51.50ms | 421.16ms |
| `600519.SH` | 沪市主板 | 45 | 3 / 3 | 42.69ms | 46.21ms |
| `300750.SZ` | 创业板 | 72 | 3 / 3 | 55.03ms | 67.21ms |
| `000524.SZ` | 深市、含长期停牌样本 | 43 | 3 / 3 | 48.55ms | 51.51ms |

`000001.SZ` 首轮的 421.16ms 是本次唯一明显较慢的请求，后两轮为 51.50ms、35.79ms。这表明任务实现必须复用客户端连接，并在任务审计中分别记录建连、请求、解析和写库耗时。

不能用上述中位数直接推算 5,206 只全量耗时：全量运行还会受到请求限速、源端波动、失败重试、序列化和 ClickHouse 写入的影响。正式建立 bootstrap 任务前，需要另做 300 只和全池基准测试。

## 接口参数

底层方法签名实测为：

```python
client.xdxr(symbol="", **kwargs)
```

项目适配器签名为：

```python
MootdxSource.fetch_xdxr(symbol: str) -> pandas.DataFrame
```

| 参数 | 层级 | 要求 | 说明 |
| --- | --- | --- | --- |
| `symbol` | 项目适配器 | `000001.SZ`、`600519.SH` 这类带市场后缀代码 | 适配器会提取六位代码后请求源端。市场后缀仍必须保留在项目数据模型中，避免沪深代码语义丢失。 |
| `symbol` | Mootdx 底层 | 六位纯数字代码，如 `000001` | 当前标准行情接口通过代码路由市场。 |
| `**kwargs` | Mootdx 底层 | 暂不使用 | 当前未验证任何额外参数的业务含义或兼容性，不应写入生产任务配置。 |
| `rate_limit` | `MootdxSource` 构造参数 | 秒 | 项目适配器在两次请求之间施加的最小间隔，不是源端接口参数。 |
| `timeout` | `MootdxSource` 构造参数 | 秒 | 建连和读取超时；本次为 `15`。 |
| `bestip` / `server` | `MootdxSource` 构造参数 | 二选一按需使用 | `bestip` 会重新测速选服务器；固定服务器便于可复现基准。 |

接口没有可用的起止日期参数，因此每次成功请求返回该标的可提供的完整事件历史，而不是指定日期的增量事件。这是后续任务设计和性能评估的关键约束。

## 返回值与字段

返回值是 `pandas.DataFrame`。本次四个样本字段集合一致：

```text
year, month, day, category, name, fenhong, peigujia,
songzhuangu, peigu, suogu, panqianliutong, panhouliutong,
qianzongguben, houzongguben, fenshu, xingquanjia
```

| 字段 | 本次类型 | 作用 | 当前处理规则 |
| --- | --- | --- | --- |
| `year`, `month`, `day` | `int64` | 事件日期组成部分 | 合成为 `event_date`；必须校验为有效日期。 |
| `category` | `int64` | 源端事件类别编码 | 不以数字硬编码业务含义；与 `name` 一起保存。 |
| `name` | `str` | 源端类别名称 | 本次样本包含“除权除息”“股本变化”“送配股上市”“非流通股上市”“转配股上市”。 |
| `fenhong` | `float64` | 分红数值 | 保留原值，单位和复权公式需与现有源逐事件对账。 |
| `peigujia` | `float64` | 配股价格 | 保留原值，不能遗漏。 |
| `songzhuangu` | `float64` | 送转股数值 | 保留原值。 |
| `peigu` | `float64` | 配股数值 | 保留原值。 |
| `suogu` | `object` | 缩股数值 | 样本全为空；不能在落库时转换为 `0`。 |
| `panqianliutong`, `panhouliutong` | `float64` | 变动前后流通股本 | 仅在部分事件存在；单位待双源对账确认。 |
| `qianzongguben`, `houzongguben` | `float64` | 变动前后总股本 | 仅在部分事件存在；单位待双源对账确认。 |
| `fenshu`, `xingquanjia` | `object` | 源端扩展字段 | 当前样本全为空；原样保留在 `raw_json`，暂不进入复权计算。 |

空值是源端事实，不是零值。首版独立表应使用 `Nullable` 字段或原始 JSON 保留空值语义。

## 数据示例

`600519.SH` 的首条样本：

```json
{
  "year": 2002,
  "month": 7,
  "day": 25,
  "category": 1,
  "name": "除权除息",
  "fenhong": 6.0,
  "peigujia": 0.0,
  "songzhuangu": 1.0,
  "peigu": 0.0
}
```

该样本仅证明字段存在和返回稳定；它不能单独证明每个字段的单位、每股/每十股口径或适用的复权公式。
