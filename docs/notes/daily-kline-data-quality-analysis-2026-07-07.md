## 关键结论

1. **数据源限制**：当前日线数据主要来自腾讯/新浪等免费 API，历史只能回溯到 2020 年
2. **严重缺口股票少**：仅 5 只有明显数据缺失（<95% 完整度），可能是历史停牌或数据源问题
3. **轻微缺口普遍**：大多数股票有少量缺失（99%-99.9% 完整度），可能是偶发停牌或数据同步失败
4. **截断问题**：541 只股票数据不是最新的，需要检查同步任务是否覆盖

## 是否需要增加数据源？

**取决于需求**：

- **如果只是做近期策略研究（2020 年以后）**：当前数据质量足够，严重缺口的 5 只股票可以手动修复
- **如果需要更长的历史回测（2015年以前）**：需要引入付费数据源（如 Tushare Pro、Wind、Choice）
- **如果需要完整覆盖所有股票**：当前 90% 更新率还行，但可以优化同步任务确保 100% 覆盖

## 附录：查询 SQL

### 整体概况查询

```sql
SELECT
    toString(min(date)) AS first_date,
    toString(max(date)) AS last_date,
    count(DISTINCT date) AS total_days,
    count(DISTINCT symbol) AS total_symbols,
    count() AS total_rows
FROM daily_kline
```

### 数据天数分布查询

```sql
SELECT
    CASE
        WHEN days >= 1500 THEN '1500+ 天 (完整)'
        WHEN days >= 1000 THEN '1000-1499 天'
        WHEN days >= 500 THEN '500-999 天'
        WHEN days >= 100 THEN '100-499 天'
        ELSE '< 100 天'
    END AS bucket,
    count() AS stock_count
FROM (
    SELECT symbol, count(DISTINCT date) AS days
    FROM daily_kline
    GROUP BY symbol
)
GROUP BY bucket
ORDER BY min(days)
```

### 内部缺口分析查询

```sql
WITH
    symbol_range AS (
        SELECT symbol, min(date) AS first_date, max(date) AS last_date
        FROM daily_kline
        GROUP BY symbol
        HAVING count(DISTINCT date) >= 1000
    ),
    expected_days AS (
        SELECT r.symbol, count(t.date) AS expected
        FROM symbol_range AS r
        INNER JOIN trade_calendar AS t
            ON t.date >= r.first_date AND t.date <= r.last_date
        GROUP BY r.symbol
    ),
    actual_days AS (
        SELECT symbol, count(DISTINCT date) AS actual
        FROM daily_kline
        GROUP BY symbol
    )
SELECT
    r.symbol,
    toString(r.first_date) AS first_date,
    toString(r.last_date) AS last_date,
    a.actual,
    e.expected,
    e.expected - a.actual AS gap_days,
    round(a.actual / e.expected * 100, 2) AS pct
FROM symbol_range AS r
LEFT JOIN actual_days AS a ON r.symbol = a.symbol
LEFT JOIN expected_days AS e ON r.symbol = e.symbol
WHERE a.actual / e.expected < 0.95
ORDER BY pct ASC
```
